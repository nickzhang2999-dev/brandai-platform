# BrandAI AI 图片生成平台｜UI 视觉规范文档 v0.1

## 一、整体设计定位

BrandAI 的整体视觉风格定位为：高端、简约、留白、AI 原生、品牌感强的智能设计工作台。

页面不应呈现传统后台管理系统的厚重感，也不应呈现专业设计软件的复杂工具栏，而应更接近“高端品牌官网审美 + AI 助手式交互 + SaaS 产品结构效率 + 设计工作台视觉表达能力”。

## 二、设计关键词

| 关键词 | 说明 |
|---|---|
| 高端 | 页面视觉克制、精致、有品牌质感 |
| 简约 | 减少按钮、减少边框、减少复杂层级 |
| AI 原生 | 通过输入框、对话框、智能推荐表达 AI 能力 |
| 留白 | 保留充足空间，避免信息压迫 |
| 轻量 | 操作路径短，界面负担低 |
| 柔和 | 色彩、阴影、线条均不宜过重 |
| 商业摄影感 | 图片素材呈现品牌广告级视觉 |

## 三、全局布局

一期采用桌面端优先的 Web App 布局：左侧导航栏 + 右侧主内容区。

| 区域 | 建议 |
|---|---|
| 页面最小宽度 | 1440px |
| 最佳展示宽度 | 1536px / 1600px / 1920px |
| 左侧导航宽度 | 220px - 240px |
| 主内容区左右留白 | 32px - 48px |
| 页面顶部留白 | 32px - 48px |
| 卡片间距 | 20px - 32px |

左侧导航统一为：首页、Campaign 项目、品牌知识库、素材库、模板库、工作台。不展示创意灵感。

## 七、核心组件规范

### AI 输入框

AI 输入框是 BrandAI 的核心视觉符号。使用位置包括首页、品牌知识库、工作台、Campaign 项目页弹窗和素材库搜索。

视觉要求：白色或极浅紫背景、轻描边或渐变描边、圆角 28px - 32px、左侧附件入口、右侧语音图标与发送按钮、紫色轻光晕聚焦态。

### 快捷操作卡片

首页快捷操作只保留创建新 Campaign、导入品牌知识库、生成广告视觉、优化现有设计。禁用品牌一致性检查快捷入口。

### Campaign 项目卡片

卡片内容包括封面图、项目名称、状态、品牌名称、描述、标签、进度、时间和更多操作。

### 推荐品牌瀑布流卡片

首页推荐品牌采用 Masonry 瀑布流，展示品牌图、品牌名、定位、描述和查看品牌。整体气质应高端、精致、广告感强。

### 品牌知识卡片

包括 Logo 使用规范、品牌色彩、字体规范、品牌语调、视觉参考、设计规范、品牌预览、AI 知识摘要。

### 品牌套件导入

默认首屏采用 Lovart 式单列聚焦结构：居中的标题与简短说明，下方为 3×2 六类内容卡，再下方放置横跨整行的完整品牌手册 PDF 虚线上传区。卡片点击态使用紫色描边和浅紫背景，不通过位移动画造成跳动；所有上传卡具备键盘焦点态。上传后在原区域下方展示稳定高度的阶段进度，依次说明逐页读取、视觉资产提取、六类草稿整理。手动文字规则默认收起为次级入口，不与 PDF 主路径争夺视觉注意力。

### 素材卡片

展示缩略图、格式、文件名、标签、尺寸、日期和收藏状态。选中状态采用紫色描边和轻阴影。

### 工作台画布

工作台采用左侧大图画布 + 右侧 AI 提示词编辑区。左侧占页面主体，底部展示生成变体；右侧展示提示词、关键词、品牌约束、参考素材和提交制作。

## 八、页面专项规范

首页：中央问候语 + AI 输入框 + 四个快捷操作 + 近期 Campaign 横向卡片 + 推荐品牌瀑布流。

Campaign 项目页：标题 + 搜索筛选 + 项目卡片列表 + 右侧 AI 项目摘要。

品牌知识库：顶部 AI 共创输入区 + 上传入口 + 品牌核心知识卡片 + AI 知识摘要。

素材库：标题 + 搜索筛选 + 统计卡片 + 素材网格 + 素材详情侧栏。

工作台：顶部项目路径 + 左侧大画布 + 底部变体 + 右侧提示词编辑区。

## 九、禁用设计清单

| 禁用项 | 原因 |
|---|---|
| 大量蓝色后台表格 | 破坏高端感 |
| 密集按钮组 | 与 AI 原生交互相冲突 |
| 复杂工具栏 | 会变成传统设计软件 |
| 过多一级菜单 | 导航过重 |
| 强边框和深色阴影 | 视觉显得笨重 |
| 大面积纯紫背景 | 容易降低高级感 |
| 页面信息过满 | 不符合留白原则 |
| 创意灵感一级栏目 | 已确认移除 |
