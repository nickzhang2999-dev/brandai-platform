# prd-intake · 需求调整意图记录(append-only)

本文件夹是 `/prd` 技能的落盘地:**每条用户对产品/流程/页面的感受或调整诉求 = 一个文件**。

- 命名:`<YYYY-MM-DD>-<短slug>.md`
- 每条都经过「好问题/差问题」闸门澄清(三要素:定位 / 现象 / 期望调整)后才写入。
- 归类用 `docs/10_需求调整归因表.md` §一 的 N/S/I/O/R。
- 本文件夹是**原始事实源**(逐条);`docs/10` §三 是**汇总索引**(每行链回这里的一份记录)。

> 为什么是文件夹而不是只往一张表里塞:单条记录保留完整追问 Q&A(客观事实),
> 表只留一行结论。聚合分析看 `docs/10` §四 复盘节奏。

用法:`/prd 我觉得工作台出图后选变体那一步太卡了`
