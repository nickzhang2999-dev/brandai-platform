# BrandAI AI 图片生成平台｜开发任务拆解与排期表 v0.1

## 一、一期开发总目标

完成一个可点击、可跳转、可演示、可部署的 BrandAI 前端高保真 Demo。

一期重点不接真实后端，不接真实 AI 生图接口，优先实现五个核心页面、统一左侧导航、Mock 数据渲染、基础交互模拟、高度还原当前确认 UI 效果、同步 GitHub、部署预览和交付技术团队二次开发。

## 八、风险与应对

| 风险 | 应对 |
|---|---|
| 页面命名再次调整 | 统一在 navigation.json 和路由中管理 |
| 页面新增 | 先加占位路由，不影响核心页面 |
| 视觉细节反复调整 | 组件化 + Tailwind Token 管理 |
| 效果图太精细，前端还原成本高 | 优先还原整体布局和视觉气质 |
| 图片资源不足 | 使用占位图或 AI 生成图替代 |
| Codex 生成代码不稳定 | 分 Prompt、分模块执行 |
| 组件过度耦合 | 每个页面独立拆组件 |
| Mock 数据混乱 | 统一 mockIndex.ts 导出 |
| build 报错 | 每个阶段结束都执行 build 检查 |

## 十、当前结论

BrandAI 一期开发应优先采用前端高保真 Demo + Mock 数据 + 基础交互模拟 + GitHub 工程交付。建议先用 5-7 天完成可演示 Demo，内部评审 UI、业务流程和页面交互，确认后进入真实后端、AI 模型、素材上传和权限体系开发。
