# BrandAI AI 图片生成平台｜完整 PRD 产品需求文档 v0.1

## 一、项目背景

BrandAI 面向品牌方自用场景，解决品牌广告素材生产中沟通链路长、品牌一致性弱、素材复用难、AI 出图随机性强的问题。平台围绕 Campaign 营销项目，结合品牌知识库、素材库与 AI 工作台，完成从需求输入到广告图生成、修改、归档的闭环。

## 二、产品定位

> BrandAI 是一套面向品牌方自用场景的 AI 驱动型品牌广告素材设计平台，围绕 Campaign 营销项目，帮助用户通过品牌知识库、素材库和 AI 工作台完成标准化、高效率、可复用的广告素材生成与管理。

## 六、核心业务流程

```text
首页 → 创建 Campaign → 选择/导入品牌知识库 → 上传/选择素材库素材 → 进入工作台 → 编辑提示词/关键词/品牌约束/参考素材 → 提交制作 → 生成设计图与变体 → 修改确认 → 项目归档
```

## 七、页面需求

### 7.1 首页

首页承担 AI 输入、近期项目访问、推荐品牌展示三类功能。首页不展示当前品牌、当前项目卡片，不展示品牌一致性检查快捷模块。

### 7.2 Campaign 项目页

展示搜索、筛选、项目卡片、项目进度、AI 项目摘要和项目操作。项目操作包括继续创作、补充需求、查看项目规范、提交终审、归档项目。

### 7.3 品牌知识库

品牌知识库即当前品牌唯一的品牌套件。默认主流程对齐 Lovart：首先展示 logo、字体、颜色、设计指南、图像、品牌指南 6 个内容入口，其下提供完整品牌/VI 手册 PDF 上传区；手动文字规则作为次级入口。PDF 上传后由异步任务逐页解析文字与页面视觉，自动提取并裁切 Logo、色卡、字体样张、版式和摄影参考，合并为六类可编辑草稿。用户逐项核对并确认后，内容成为当前品牌下全部项目的生成约束。

品牌套件只采用项目顶部选择的持久应用方式；输入框 `@品牌套件` 的单次应用方式暂不支持。

页面交互以 Lovart 品牌套件为目标状态，但沿用 BrandAI 紫色语义色：左侧为 60px 全局图标栏与 260px 套件卡片栏；空白态只出现开始引导；完成态使用约 640px 编辑区、120px 素材卡、可就地编辑的套件标题和原位图片灯箱。品牌预览、规则统计、状态圆点及旧手动文字共创入口不得出现在该页面。

### 7.4 素材库

展示素材搜索、分类筛选、素材统计、素材网格、素材详情侧栏、AI 标签、AI 描述、使用记录、加入项目、设为参考、查看来源。

### 7.5 工作台

采用左侧大图画布 + 右侧提示词编辑区。包括项目路径、当前广告图、生成变体、提示词、风格关键词、品牌约束、参考素材、提交制作、额度展示和免责声明。

## 八、AI 交互设计

| 页面 | AI 参与方式 |
|---|---|
| 首页 | 理解用户输入，推荐创建项目、导入知识库、生成广告视觉 |
| Campaign 项目页 | 生成项目摘要、推荐下一步动作 |
| 品牌知识库 | 解析单张素材或完整 PDF 品牌手册，生成六类品牌规则、可追溯视觉资产和关键词 |
| 素材库 | 自动识别素材标签、生成素材描述 |
| 工作台 | 根据提示词、关键词、品牌约束和参考素材生成图片 |

## 九、弹窗与浮层

一期建议弹窗包括新建 Campaign、补充需求、查看项目规范、上传品牌资料、素材上传、加入项目、查看来源、提交制作确认、额度升级。

## 十、一期 Demo 开发范围

必须实现五个页面静态高保真还原、左侧导航跳转、Mock 数据展示、素材详情侧栏、Campaign 右侧 AI 摘要、工作台图片切换、基础弹窗与统一 UI 风格。

可模拟 AI 输入解析、图片生成、素材上传、项目创建、生成额度、通知中心和登录状态。

暂不实现真实 AI 生图、真实登录鉴权、后端数据库、文件真实上传、多租户真实隔离、复杂权限、计费、视频生成和 PSD 编辑。

## 十一、前端技术建议

React + TypeScript + Vite + Tailwind CSS + React Router + lucide-react + 本地 JSON Mock 数据。

## 十二、验收标准

1. 五个核心页面完整；
2. 左侧导航命名正确；
3. 无“创意灵感”“品牌资产”“图片设计编辑页”；
4. 首页布局符合最新确认；
5. 品牌知识库、素材库、工作台功能完整；
6. 本地 Demo 可运行、可构建、可部署。
