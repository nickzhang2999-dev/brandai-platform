# 2026 年 6 月 27 日 V0.06 修改内容统计

## 版本定义

本次修改定义为 **V0.06**。

工程包版本同步调整为 `0.0.6`，用于和本次功能定义保持一致。

## 修改目标

本次仅做产品名词与品牌套件维度定义调整，不改变现有底层路由、数据表、API key 或业务流程。

## 修改内容

### 1. Campaign 项目更名为项目

将系统中面向用户展示的 **Campaign 项目** 简化为 **项目**。

涉及范围：

- 左侧导航：`Campaign 项目` 调整为 `项目`
- 项目页标题：`Campaign 项目` 调整为 `项目`
- 创建入口：`创建新 Campaign` 调整为 `创建新项目`
- 首页近期模块：`近期 Campaign` 调整为 `近期项目`
- 工作台项目选择提示：统一使用 `项目`
- 素材加入项目、参考素材提示：统一使用 `项目`

说明：底层仍沿用 `/campaigns` 路由和既有 Project/Campaign 兼容字段，避免影响已有数据与接口。

### 2. 品牌知识库更名为品牌套件

将系统中面向用户展示的 **品牌知识库** 调整为 **品牌套件**。

涉及范围：

- 左侧导航：`品牌知识库` 调整为 `品牌套件`
- 品牌规则页面标题：`品牌知识库` 调整为 `品牌套件`
- 项目规范侧栏：`品牌知识库规则` 调整为 `品牌套件规则`
- 品牌预览与 API 提示：统一使用 `品牌套件`
- 首页快捷入口：`导入品牌知识库` 调整为 `导入品牌套件`
- README 当前版本说明同步更新

说明：底层仍沿用 `/brand-knowledge` 路由与 `BrandRule` 数据结构，避免破坏现有工程实现。

### 3. 品牌套件 6 个维度重新定义

品牌套件下属内容统一定义为以下 6 项：

| 顺序 | 维度名称 | 现有数据类型 key |
|---|---|---|
| 1 | logo | `logo` |
| 2 | 字体 | `font` |
| 3 | 颜色 | `color` |
| 4 | 设计指南 | `layout` |
| 5 | 图像 | `imagery` |
| 6 | 品牌指南 | `copy` |

页面展示顺序已同步调整为：

`logo → 字体 → 颜色 → 设计指南 → 图像 → 品牌指南`

旧展示词汇对应关系：

| 旧名称 | 新名称 |
|---|---|
| Logo 使用规范 | logo |
| 字体规范 | 字体 |
| 品牌色彩系统 | 颜色 |
| 版式规范 | 设计指南 |
| 参考图 / 素材规范 | 图像 |
| 品牌语气 / 文案 | 品牌指南 |

## 研发注意事项

- 本次是名词和展示结构调整，不涉及数据库迁移。
- `Campaign`、`brand-knowledge` 等历史路由和内部 key 暂不重命名，避免产生大范围兼容风险。
- 后续如需彻底重命名内部路由或数据模型，应作为单独版本处理，并包含迁移方案、重定向方案和回归测试。
