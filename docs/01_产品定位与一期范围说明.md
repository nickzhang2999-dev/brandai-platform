# BrandAI AI 图片生成平台｜产品定位与一期范围说明 v0.1

## 一、项目背景

品牌方在广告投放、内容营销、电商运营、社交媒体传播、线下物料设计等场景中持续产生大量视觉素材需求。传统品牌广告素材制作依赖内部设计或外包团队，存在沟通链路长、交付周期慢、素材复用难、品牌一致性难保障等问题。

传统 AI 绘图工具虽然可以快速生成图片，但生成结果随机性强，难以稳定遵循品牌 Logo、色彩、字体、产品展示、文案语调、视觉规范等品牌规则，也缺少 Campaign 项目管理、素材库、知识库和交付归档能力。

因此，本项目拟构建一套面向品牌方自用场景的 AI 驱动型品牌广告素材设计平台，通过 AI 对话、Campaign 项目管理、品牌知识库、素材库、工作台等模块，帮助品牌方快速完成广告素材创建、生成、修改、复用和归档。

## 二、产品定位

> BrandAI 是一套面向品牌方自用场景的 AI 驱动型品牌广告素材设计平台，围绕 Campaign 营销项目，帮助用户通过品牌知识库、素材库和 AI 工作台完成标准化、高效率、可复用的广告素材生成与管理。

## 三、产品关键词

| 关键词 | 说明 |
|---|---|
| AI 驱动 | 所有核心操作尽量通过 AI 对话、弹窗、提示词完成 |
| Campaign 项目制 | 所有设计工作围绕具体营销项目展开 |
| 品牌知识库 | 将品牌规范、视觉规则、文案语调、设计风格结构化沉淀 |
| 素材库 | 管理产品图、包装图、参考图、视频、文档、纹理等品牌资产 |
| 工作台 | 大图预览 + 提示词编辑 + 参考素材 + 提交制作的核心出图页面 |
| 高端简约 UI | 视觉上采用高端、留白、轻量、AI 原生的产品风格 |
| 专业交付 | 最终交付 PRD、原型图、UI 规范、开发说明和代码实现基础 |

## 四、目标用户

| 用户角色 | 典型身份 | 核心诉求 |
|---|---|---|
| 市场负责人 | 市场总监、品牌负责人、营销负责人 | 快速创建 Campaign，管理项目进度，确认最终视觉方向 |
| 品牌运营人员 | 品牌专员、内容运营、电商运营 | 输入需求、调用素材、生成广告图、修改内容 |
| 设计协同人员 | 内部设计、外部设计支持 | 参考 AI 生成结果，进一步优化视觉表现 |
| 项目确认人员 | 品牌负责人、项目负责人 | 查看项目结果，提出修改意见，确认最终交付物 |
| 平台管理员 | 企业管理员、实施人员 | 管理品牌空间、账号、权限、基础配置 |

## 五、一期核心目标

> 跑通 1-2 家品牌方客户在专属定制场景下，从 Campaign 项目创建、品牌知识库配置、素材库管理、工作台 AI 出图、修改优化到最终交付归档的完整业务闭环。

## 六、系统架构定位

本平台采用 SaaS 多租户架构逻辑。一期仅服务 1-2 家合作客户，按专属定制模式开发；后续接入更多客户后，再逐步升级为标准 SaaS 多租户运营形态。

## 七、产品交互原则

| 原则 | 说明 |
|---|---|
| AI 对话优先 | 用户主要通过输入描述、粘贴 brief、上传资料与 AI 对话完成操作 |
| 弹窗 / 对话框优先 | 复杂操作尽量通过轻量弹窗完成，不频繁跳转页面 |
| 减少按钮堆叠 | 避免传统软件中大量按钮、选项、工具栏、层级菜单 |
| 单界面完成核心流程 | 首页、Campaign、品牌知识库、素材库、工作台各自承担明确任务 |
| 保留必要功能 | 业务逻辑不删减，仅优化 UI 和交互形式 |
| 轻量化、智能化 | 让系统主动理解用户意图、整理内容、推荐下一步动作 |
| 视觉高级感 | 采用高端简约、留白、柔和阴影、浅色背景、蓝紫点缀 |

## 八、一期固定页面范围

## 十、一期核心业务流程

```text
用户进入首页
↓
输入广告素材需求 / 点击创建新 Campaign
↓
通过弹窗或 AI 对话创建 Campaign 项目
↓
选择或导入品牌知识库
↓
上传项目相关素材至素材库
↓
进入工作台
↓
输入提示词、选择参考素材、调用品牌约束
↓
提交制作
↓
生成图片结果与变体
↓
用户继续修改或确认
↓
素材进入项目归档 / 交付
```

## 十一、一期不纳入范围

| 多级审批流 | 暂不做 |
| 完整商业计费系统 | 暂不做 |

| PSD 深度编辑 | 暂不做 |
| 泛网络素材抓取 | 不做 |
