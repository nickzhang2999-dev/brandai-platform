> 目标：把 prd_agent「视觉创作」编辑器的**右侧对话功能（图生图 / 多图生图）**、**水印**、**系统提示词**迁移到 BrandAI 工作台，交互对标 Lovart 的 AI 设计师面板。

## 二、prd_agent 两个已知 bug 的根因与规避策略（本轮**不搬运**）

### Bug 1 · 多图生图报「Vision API 响应格式不支持」
- **根因**（`OpenAIImageClient.cs:1506+`）：多图走通用 Vision 分支（`/chat/completions`），响应解析只认 `choices[0].message.content` 字符串里的图片；gemini 经 OpenAI 兼容网关返回的 `message.images[]` / 多模态数组一概不识别 → 抛 1734 行错误。单图路径已兼容这些形态，所以「单图 OK、多图挂」。
- **BrandAI 规避**：**不采用 chat/completions Vision 路线**。多图统一走 OpenAI `/images/edits` multipart（多个 `image[]` part），provider 层 `generate_with_references` 已有实现与响应解析（`data[].b64_json|url`），单图/多图同一条代码路径、同一个解析器，结构上消除「单图多图两套解析」的分叉。pytest 断言多图请求把每张参考都作为独立 multipart part 发出。

### Bug 2 · 对话文本框信息冗余（用户可见消息混入文件名/引用块）
- **根因**（`imageRefResolver.ts:169` `buildRequestText`）：前端把 `【引用图片（按顺序）】\n- @img1: 76a970….png` 拼进 prompt；用户不打字时 `display || reqText` 回退 + 后端 `UserMessageContent ?? prompt` 回退，把整段模型 prompt（含文件名）落库成用户可见消息。
- **BrandAI 规避**：**展示层与模型层物理分离**。
  1. 用户可见文本 = `chatContext.displayText`（用户敲的原文，可为空），引用图以 chip 缩略图展示，**任何路径都不把 URL/文件名/引用块写进可见文本**；
  2. 模型 prompt 由 worker/AI 服务从结构化字段（`sellingPoint` + `imageInputs`）组装，只存在于 `GenerationVersion.params.prompt`（调试可见，不进会话气泡）；
  3. 契约测试锁死：`chatContext.displayText` 原样往返，无任何自动拼接。

## 三、技术设计（BrandAI 侧）

```
ChatPanel(工作台右侧新 Tab「AI 设计师」)
  输入框(Enter 发送) + 引用 chips(点历史出图/素材缩略图添加, ≤8, 有序)
    │  POST /api/workspaces/[wsId]/generations
    │  CreateGenerationInput + { imageInputs:[{kind:VERSION|ASSET,id}], chatDisplayText }
    ▼
route.ts: 归属校验(版本/素材均须本 workspace, IDOR) → Generation 落库(chatContext Json)
    → enqueue generate job(imageInputs 已解析成 {url,sourceHint})
    ▼
generate.worker.ts: imageInputs → aiConstraints.referenceImages
    (mode=STRICT, note="IMAGE_INPUT:{序号}", polarity=positive)
    + AppSetting.imageSystemPrompt → GenerateRequest.systemPrompt
    ▼
apps/ai /v1/generate: strict_refs 全量(≤8) → generate_with_references
    (多张 image[] multipart → /images/edits)；IMAGE_INPUT 用「按指令合成/改绘」
    prompt 模板，与遗留「锁定素材」模板区分
    ▼
出图 → 水印叠加(既有) → GenerationVersion → 前端轮询同一 Generation 历史 → 会话流更新
```

关键决策：
1. **不新增消息表**。会话流 = `Generation` 历史的投影（`chatContext` 存展示文本与引用），服务端权威、刷新/深链天然可恢复（符合 §0.7）。
2. **`imageInputs` 是新语义**，与 V0.0.12 的「素材 STRICT→水印叠加」「模板参考→INSPIRATION」互不干扰，全部 frozen-additive。
3. 契约两侧同步：`ai.ts`/`api.ts`/`entities.ts` ↔ `schemas.py`；顺手修掉 `schemas.py::ReferenceImage` 重复定义的 `mode` 字段。

## 六、明确不做（本轮范围外，含归因）

- 对话面板 per-request 模型选择（BrandAI 单 provider 架构，需先有多 provider 池）— R（优化提议，后续版本）。
- prd_agent 侧 bug 修复（迁移目标是 BrandAI，源仓库另行处理）。
- 局部涂抹/蒙版进对话流（工作台已有 INPAINT 表单模式，不重复建设）。
- 新增 Conversation/Message 表（用 Generation 投影即可，避免双事实源）。
