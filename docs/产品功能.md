## 一、产品定位

BrandAI 是一个以“品牌”和“项目”为中心的视觉 AI 生成平台。用户先沉淀品牌套件，再创建项目、组织素材，通过 AI 工作台生成和编辑图片，最后完成审阅、终选、导出与归档。

当前产品的核心价值是：

1. 将分散在 VI 手册、Logo、色板、字体、版式和图像样例中的品牌要求结构化。
2. 在生成图片时自动调用当前品牌套件，不让用户重复选择 Logo、颜色或规则。
3. 提供从需求输入、生成、画布编辑、品牌合规到交付归档的完整闭环。

## 四、核心概念

| 产品术语 | 系统含义                                                               |
| -------- | ---------------------------------------------------------------------- |
| 品牌     | 数据层为 `BrandWorkspace`，是规则、素材、项目和成员隔离的根容器        |
| 项目     | 数据层为 `Project`，页面文案统一使用“项目”；承载一次 Campaign/视觉任务 |
| 品牌套件 | 品牌 Logo、颜色、字体、设计指南、图像风格及底层文案规则的集合          |
| 品牌规则 | `BrandRule`，具有类型、强度和 DRAFT/CONFIRMED/REJECTED 状态            |
| 素材     | 用户上传、网站采集或系统生成的可复用文件                               |
| 模板     | 作为 AI 灵感参考的 `TEMPLATE` 类型素材，不等同于确定性水印             |
| 生成图   | AI 生成版本的独立列表，同时可镜像为 `GENERATED` 素材                   |
| 生成任务 | 一次服务端异步任务，状态为 PENDING/RUNNING/SUCCEEDED/FAILED            |
| 生成版本 | 一次生成或改图得到的具体图片，可审阅、终选、下载和回流素材库           |

用户端左侧导航包含 8 个入口：

| 页面      | 路径               | 核心作用                                              |
| --------- | ------------------ | ----------------------------------------------------- |
| 首页      | `/`                | 品牌概览、AI brief 立项、快捷入口、近期项目、推荐品牌 |
| 项目      | `/campaigns`       | 项目创建、检索、筛选、摘要、归档与进入创作            |
| 品牌套件  | `/brand-knowledge` | 新建/切换套件、导入 VI、管理品牌内容和启用状态        |
| AI 工作台 | `/workspace`       | 对话生图、开放画布、图生图、改图和终选                |
| 生成图    | `/generated`       | 独立查看和筛选 AI 生成结果                            |

## 六、端到端业务流程

1. 用户登录并进入当前品牌。
2. 新建或切换品牌套件。
3. 上传整本 PDF VI 手册，或上传 Logo、色板、字体样张、版式和图像文件。
4. 系统异步解析文字、页面和图片，生成规则草稿及品牌素材。
5. 用户确认规则，并开启“应用到项目”。
6. 用户新建项目，填写名称、简介和渠道等信息。
7. 用户在素材库上传或选择项目素材，也可选择模板作为灵感参考。
8. 用户进入 AI 工作台，用自然语言描述创作目标，可附带最多 8 张有序图片引用。
9. 系统自动判断品牌套件：有已确认规则则强制应用；没有规则则自由创作。
10. Worker 调用真图像模型，随后自动合成品牌 Logo、执行合规检查并保存生成版本。
11. 用户在开放画布继续排版、图生图、局部重绘、扩图、换背景或改色。
12. 用户提交审阅，Owner/Reviewer 批准或驳回。
13. 用户设为终稿，下载单图或导出项目 ZIP，最后归档项目。

## 七、模块功能说明

### 7.1 账号、认证与导航

- 支持邮箱密码登录，以及配置后的 GitHub/Google OAuth。
- 未登录用户会被服务端和边缘中间件阻止访问用户页面及接口。
- 用户可在账号设置修改昵称和密码。
- 用户菜单区分个人设置与管理后台；只有管理员可见管理入口。
- 顶部通知中心从真实生成任务和异步任务终态派生通知。
- 当前品牌由服务端 cookie/成员关系解析，刷新和深链保持一致。

### 7.2 首页

- 显示基于真实用户昵称的问候语。
- AI brief 输入可异步拆解卖点、场景、风格和场景类型，并预填工作台。
- 提供创建项目、导入品牌套件、生成视觉和优化设计的快捷入口。
- 展示最近项目及真实项目状态/进度。
- 推荐品牌仅从用户有权访问的品牌范围中产生，不跨租户泄漏。

### 7.3 项目

- 创建项目：名称、描述、渠道等信息进入真实数据库。
- 支持名称搜索、阶段筛选、近 7/30/90 天筛选，以及按最近、名称、进度排序。
- 项目卡展示封面、状态、描述、标签、渠道和进度。
- AI 项目摘要由异步 VLM 任务生成并写入项目。
- 项目操作包含继续创作、补充需求、查看规范、提交终审和归档。
- 项目状态包含草稿、进行中、已完成；归档以独立 `archivedAt` 标记保存。

### 7.4 品牌套件

#### 页面交互

- 页面采用 Lovart 式两栏结构：60px 全局图标栏 + 260px“我的品牌套件”卡片栏。
- 套件列表支持新建、切换、就地改名和删除。
- 套件卡缩略图自动使用 Logo 分区的第一张图片。
- 图片不跳转新页面，直接在当前页灯箱中预览、缩放和下载。
- 空白套件只显示“开始使用你的品牌套件”和导入入口。
- 已有内容后，空白引导消失，改为品牌标题、导入条和紧凑内容架。

#### 品牌内容

页面展示 5 个不重复分区：

1. Logo
2. 颜色
3. 字体
4. 设计指南
5. 图像

底层仍可保存文案、语调和禁用表达等规则，但不再重复显示一个与“设计指南”内容重叠的“品牌指南”区块。

#### 导入与解析

- 支持上传整本 PDF 品牌手册，最多处理 120 页。
- 系统逐页提取精确文字、渲染页面并分批调用 VLM。
- 自动识别 Logo、字体、颜色、设计指南、图像及底层品牌语言规则。
- 自动裁切模型选中的 Logo、色卡、字体样张、版式和摄影区域，保存为隔离的 `BRAND_KIT` 证据素材。
- 视觉批次遇到裸 5xx 或网络超时时会降级保留已成功页面与 PDF 文字，不让整本手册失败。
- 新规则先进入 DRAFT，用户确认后进入 CONFIRMED 并参与生成。

#### 应用逻辑

- 页面开关文案为“应用到项目”。
- 开启：当前品牌套件可被当前品牌下的项目自动调用。
- 关闭：项目生成、合规检查和导出链路都不能调用该套件规则。
- 当前采用项目顶部选择品牌套件后自动约束项目内全部生成。
- 暂不支持在输入框中通过 `@品牌套件` 仅约束单次生成。
- 品牌套件页面不再包含“品牌预览”、手动文字共创或快捷提示词入口。

### 7.5 素材库

- 支持真实文件上传、文件夹新建、筛选、移动和删除。
- 支持名称搜索、素材类型筛选、收藏筛选和统计。
- 素材详情展示来源、尺寸、标签、AI 描述和使用记录。
- 可调用真 VLM 自动生成 AI 标签和描述。
- 可把素材加入项目，或设为项目参考素材；关系保存在服务端，跨设备可恢复。
- 生成图会镜像为 `GENERATED` 素材，但“素材库”“模板库”“生成图”仍保持三类入口分离。
- 不可用或已弃用素材不能继续参与生成。

### 7.6 模板库

- 模板以 `TEMPLATE` 素材形式保存。
- 模板用于 AI inspiration，影响构图和视觉方向。
- 模板不是原样水印，也不保证逐像素复现。
- 从模板进入工作台后，模板引用随项目生成请求保存。

### 7.7 AI 工作台

#### 页面结构

- 左侧为开放世界画布，右侧为唯一 AI 设计师对话面板。
- 顶部显示当前品牌、当前项目和“品牌套件自动应用 · N 条规则”。
- 不提供 Logo、颜色、字体或品牌规则的手动选择按钮。

#### 对话与生成

- 用户通过自然语言直接描述需求。
- 支持纯文本生图、单图生图和多图生图。
- 一次最多 8 张有序图片引用，可来自生成版本或素材。
- 图片引用通过画布选择、上传、拖放或粘贴进入对话上下文。
- 对话历史保存用户原文及引用图片，显示文本与模型实际 prompt 分离。
- 管理员可配置全局图像系统提示词，在每次生成时自动注入。
- 图像模型当前固定为 `gpt-image-2`，不要擅自回退旧模型。

#### 自动品牌约束

- 没有已确认品牌规则：进入 `FREE/direct`，模型按用户需求自由创作。
- 有已确认品牌规则：进入 `BRANDED/branded_direct`。
- 品牌规则、色板和主 Logo 在用户 brief 之前进入模型边界。
- 主 Logo 同时作为模型视觉上下文；模型被要求预留安全区，不能发明替代 Logo。
- 模型返回后，Worker 使用品牌套件原始 Logo 素材进行确定性合成，确保 Logo 可见且不被模型重绘。
- 最终图片进入自动合规检查。
- 每个生成版本记录约束模式、实际规则数量、参考图、Logo 资产和合成方式。

#### 开放画布

- 支持图片、矩形、圆形、文字和生成器区域等元素。
- 支持单选、多选、框选、拖拽、四角缩放、删除和方向键微调。
- 支持触控板平移、捏合缩放、鼠标/快捷键缩放、空格临时手型。
- 支持图片上传、拖放、剪贴板粘贴和素材/模板进入画布。
- 上传图片先显示本地预览，再在后台持久化，并显示同步状态。
- 画布内容、相机位置和被删除的版本列表保存在服务端 `ProjectCanvas`，刷新和换设备可恢复。

#### 改图与交付

- 选中生成图后显示贴近图片的快捷操作条。
- 支持局部重绘、扩展、换背景、改色、改文字、增加或移除元素。
- 局部重绘使用真实蒙版并调用图像编辑接口。
- 改图结果作为子版本保存，不覆盖原图。
- 支持历史出图回看、具体生成图深链和变体切换。
- 支持提交审阅、批准/驳回、设为终稿、单图下载和项目 ZIP 导出。

### 7.8 生成图

- 生成图有独立一级入口，不与普通素材混在默认列表。
- 支持按名称/提示词和项目维度检索。
- 可从通知、队列或深链直接进入具体生成版本。
- 每个版本保留实际尺寸、生成参数、合规报告、父版本、审阅状态和终稿状态。

### 7.9 成员协作与审阅

- Owner 可邀请已注册用户加入品牌空间。
- 支持 OWNER、EDITOR、REVIEWER、VIEWER 等角色语义。
- Owner 可调整成员角色或移除成员。
- Editor 可提交版本；Reviewer/Owner 可批准或驳回并填写意见。
- 协作者的下载和导出遵守最终版/已批准版本的发布策略。

### 7.10 管理后台

- 独立管理布局，不与普通用户侧栏混用。
- 包含平台设置、用户管理、订阅额度、全部空间、用量看板和运行日志。
- 可配置图像/VLM provider、模型、加密 API Key、对象存储和图像系统提示词。
- 可测试 provider 连通性，配置遵循数据库优先、环境变量兜底。
- 可启停用户、控制开放注册、调整各套餐的日/月额度和品牌上限。
- API Key 使用 AES-256-GCM 加密，任何接口都不能把明文 Key 返回前端。

## 八、全局业务规则

| 规则ID | 规则                                                                                                 |
| ------ | ---------------------------------------------------------------------------------------------------- |
| GR-01  | 品牌、项目、规则、素材和生成记录全部必须在 workspace 范围内查询并校验成员身份。                      |
| GR-02  | 会被刷新或分享的当前品牌、项目和生成图状态必须由 cookie、URL 或数据库保存，不能只放 `localStorage`。 |
| GR-03  | 任何 AI 或外部慢调用必须进入异步 Worker；HTTP handler 只鉴权、快检、落 PENDING、入队并返回 202。     |
| GR-04  | 已确认且已启用的品牌套件自动约束项目生成；关闭套件或没有确认规则时不得假装已应用。                   |
| GR-05  | Logo 精确呈现采用“模型理解 + 原始素材确定性合成”双层方案，不要求模型自行重绘 Logo。                  |
| GR-06  | FORBIDDEN 规则和禁用词在生成前检查；最终图片继续做视觉合规检查。                                     |
| GR-07  | 验收必须使用真 provider、真 API、真 DB；mock 只允许本地交互或契约测试。                              |
| GR-08  | 图片模型固定使用 `gpt-image-2`，除非产品负责人明确改变铁律。                                         |
| GR-09  | 生成、改图、解析和识别任务必须允许刷新、离开页面后继续完成，并提供失败和超时出口。                   |
| GR-10  | 用户端不得重新长出 Logo、颜色、字体或品牌规则手动选择按钮。                                          |

## 九、明确边界与暂不支持项

以下内容不是当前缺陷，除非产品负责人重新立项，否则不要擅自补：

- 输入框 `@品牌套件` 的单次约束模式。
- 品牌套件独立“品牌预览”模块。
- 品牌套件内手动文字共创和快捷提示词。
- 商业支付闭环和完整账单系统。
- 未注册邮箱的邀请—注册令牌闭环。
- 视频生成、PSD 深度编辑和泛网络素材抓取。
- 视觉创作来源项目中的手绘板和专属移动端重构。

当前工作台仍存在但不阻断主流程的体验差异：

- 尚无完整图片右键菜单。
- 输入框尚不支持纯键盘 `@图片` 提及选择。
- 上传前大图压缩仍可继续优化。
- 部分高级图层调序和顶部自动排列按钮仍可继续完善。

可将下面内容连同本文件与 [产品文档](./产品文档.md) 一起交给新 AI：

> 你正在开发 BrandAI V0.0.18。先阅读 `docs/产品功能.md`、`docs/产品文档.md`、`CLAUDE.md` 和 `README.md`。当前基线位于 `codex/brandai-v0.0.18`，不要假设已合并 main。任何修改必须保持品牌套件自动应用，不在工作台新增 Logo/颜色/规则选择按钮；慢 AI 调用必须进 Worker；所有查询必须做 workspace 成员校验；契约需同步 Zod 与 Pydantic；验收不得使用 mock。先核对现状与目标，再开始修改。
