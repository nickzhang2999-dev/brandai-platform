## 3. 首页：近期 Campaign 封面

### 已修改

- 首页只请求最近 8 个 Campaign。
- 服务端为这 8 个 Campaign 查询各自最新的 `GenerationVersion` 图片地址。
- 首页卡片优先展示该图片；没有任何生成图时，继续使用项目 ID 生成的默认紫色渐变。
- 图片 URL 无法加载时，前端自动回退到默认渐变，避免出现破损图片。

### 研发实现

| 位置 | 说明 |
| --- | --- |
| `apps/web/src/lib/generations.ts` | `listWorkspaceProjects` 新增 `take`、`includeLatestCover` 选项，按项目解析最新版本图片 |
| `apps/web/src/app/api/workspaces/[wsId]/projects/route.ts` | 支持 `?latestCover=1` 参数；该模式限制为最近 8 个项目 |
| `apps/web/src/app/(brandai)/page.tsx` | 使用真实 `coverImage` 渲染封面，并提供图片错误回退 |

### 行为边界

- 此处的“最新”以 `GenerationVersion.createdAt` 为准。
- 该读取逻辑只在首页启用，不会让其他项目列表承担额外的封面查询开销。
- `Project.coverImage` 仍保留为兼容兜底字段；首页优先使用真实的最新生成图。

## 5. 品牌知识库：多品牌与内容管理

### 5.1 核心概念

现有 `BrandWorkspace` 被作为“一个品牌知识库”的物理边界使用。每个 workspace 已天然隔离：

- 品牌规则 `BrandRule`
- 素材 `Asset`
- Campaign 项目 `Project`
- AI 生成任务及其版本

本轮没有新建重复的数据表，而是把已有的多 workspace 能力暴露为产品层的“品牌知识库”切换和创建能力。

### 5.2 新建与切换知识库

- 左侧导航新增“当前品牌知识库”下拉框。
- 品牌知识库页增加当前知识库选择器和“新建知识库”入口。
- 新建时填写知识库名称，行业为可选字段。
- 切换后，当前页面、Campaign 页面、素材库和 AI 工作台使用新的 `wsId` 请求数据。
- 当前选择写入浏览器 `localStorage`，刷新后会恢复；无法读取列表时仍回退到服务端解析的默认品牌。

### 5.3 统一内容入口

已移除原来的六个独立上传卡：Logo、品牌描述、色值规范、参考图、素材、文案规范。

替换为“添加知识内容”统一入口：

| 输入方式 | 处理方式 | 产物 |
| --- | --- | --- |
| 文本描述 | 创建当前规则类型的草稿规则 | 可编辑的 `DRAFT` 规则卡 |
| 图片 | 上传至当前知识库，再调用规则识别任务 | AI 识别生成的草稿规则卡 |
| PDF | 上传为 VI 文档，再调用手册解析任务 | AI 解析生成的草稿规则卡 |

规则类型中明确提供“参考图 / 素材规范”，用于补足素材规范的录入入口。

### 5.4 规则生命周期

每个规则卡片新增以下操作：

| 操作 | 行为 |
| --- | --- |
| 增加同类 | 回到统一内容入口，并预选当前规则类型 |
| 编辑 | 修改规则说明、约束强度和结构化 JSON 内容 |
| 保存 | 通过规则 PATCH 接口更新内容 |
| 启用 | 将 `DRAFT` 或 `REJECTED` 状态切换为 `CONFIRMED` |
| 禁用 | 将已启用规则从 `CONFIRMED` 切换为 `REJECTED` |
| 删除 | 调用规则 DELETE 接口永久删除 |

禁用规则仍保留在知识库界面，能够继续编辑、重新启用或删除，不再是不可恢复的终态。

### 5.5 最新内容优先

AI 出图读取已启用规则时，规则顺序由创建时间正序改为最近更新时间倒序。新建或刚编辑并启用的规则会优先进入生成约束，旧规则仍作为同一知识库的补充内容保留。

### 5.6 研发实现

| 位置 | 说明 |
| --- | --- |
| `apps/web/src/app/(brandai)/brand-context.tsx` | 读取、切换、创建品牌知识库；维护当前 `wsId` |
| `apps/web/src/app/(brandai)/brand-sidebar.tsx` | 增加全局知识库切换器；通知和队列使用当前知识库 ID |
| `apps/web/src/app/(brandai)/layout.tsx` | 将侧栏置于可切换的品牌上下文中 |
| `apps/web/src/app/(brandai)/brand-knowledge/page.tsx` | 新建知识库、统一输入、图片/PDF 解析、规则编辑及启停删除交互 |
| `apps/web/src/lib/rules.ts` | 已启用规则按 `updatedAt desc` 返回，保证最新内容优先 |

## 8. 后续研发注意项

1. 当前“品牌知识库”复用 `BrandWorkspace`，无需新迁移；后续若要单独记录知识库版本、审批流或一个品牌下多套知识库，再评估拆分独立实体。
2. 统一输入中的文本目前创建规则草稿；图片和 PDF 才走异步 AI 识别/解析。若需要“文本也必须由 AI 自动拆分成多条规则”，需要新增文本解析接口与 worker。
3. 最新规则优先通过排序实现，旧的已启用同类规则不会自动禁用。产品可继续决定是否需要“一类规则只能有一个默认版本”的显式默认标记。
4. 在可联网的开发环境中，应执行 `pnpm --filter @brandai/web typecheck`，并补充多知识库切换、图片/PDF 上传、规则状态恢复的端到端测试。
