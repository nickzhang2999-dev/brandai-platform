## 比对员缺口原文（按用户影响排序）

与 prd_agent 视觉创作（AdvancedVisualAgentTab.tsx，9411 行）对照，按用户影响排序：

1.（影响最大）上传不是「先本地预览再异步同步」：prd_agent onUploadImages 先 compressImageForCanvas 压缩、FileReader 读 dataURL **立即落画布**（status:'done'+syncStatus:'pending'），再后台 uploadVisualAgentWorkspaceAsset 回填 assetId/公网 URL/synced（Tab:4952-5081）；BrandAI uploadFiles 逐张 `await onUploadImage(file)` 等服务器上传完才建 item（OpenCanvas.tsx:826-849 + page.tsx:1091-1128），慢网/大图时画布数秒零反馈、无占位 tile、无压缩。功能可用但体感明显差于源实现。
2. 浮动操作条不贴图：prd_agent 快捷操作栏在**选中图上方居中**、快捷编辑输入框在**图下方居中**，世界坐标跟随图片与 transform、scale(var(--invZoom))（Tab:6818-6906）；BrandAI 是画布顶部固定居中一条合并工具条（ops+指令输入+终选/审阅，OpenCanvas.tsx:1591-1664）。功能等价（op+指令+inpaint 都在），形态不一致——图在画布下方/边缘时操作条与图分离。
3. 无图片右键菜单：prd_agent 有 contextmenu 菜单承载图层等操作（Tab:2366-2373、6063-6097、8620-8626）；BrandAI 无任何 onContextMenu（图层操作退化到左侧工具栏按钮）。习惯不一致，非阻断。
4. 无 @ 提及：prd_agent composer 是 Lexical RichComposer + ImageMentionPlugin（@ 触发图片下拉插 chip，RichComposer/ImageMentionPlugin.tsx:21-66）；BrandAI 是裸 contentEditable，插 chip 只能靠点选画布图/粘贴 token，键盘党无法纯输入框内引用图。
5. composer 几何小差异：prd_agent 默认 minHeight 40 / maxHeight 150（TwoPhaseRichComposer.tsx:84-85，随内容长高）；BrandAI 固定 minHeight 96 / maxHeight 132 / 14px（ChatPanel.tsx:734-739）。chip 解剖、待确认徽标、底栏（尺寸/模型/圆发送）均已 1:1，此项属可接受偏差。
6. prompt 稀释——**已核验无此问题**（见 existing 第 17 条）：chat 来源 prompt=纯 brief+图像输入，场景模板/品牌规则/promptAdditions/品牌参考图全部剥离；仅 imageSystemPrompt（管理员显式配置）与 negativePrompt/硬禁令/预检等安全底线保留，属设计内。
7. 次要：无上传前大图压缩（prd_agent compressImageForCanvas）；对话模型 chip 硬编码 "gpt-image-2" 文案（ChatPanel.tsx:893，铁律模型，属可接受）。
