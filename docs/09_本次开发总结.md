### 4. PR #1 Codex 评审：安全/数据/正确性 31 条全修
分类（详见 git log）：
- **SSRF 全维度**（web `lib/ssrf.ts` + AI `app/ssrf.py`）：初始 URL + 逐跳重定向 + IPv6 映射（`::ffff:7f00:1`）+ 仅放行 `data:image/*`；被拦 URL 不再转发给 VLM；raw 代理非图片强制 `octet-stream + attachment + nosniff`。
- **凭据暴露**：移除 `cds-compose.yml` 的 `cds.path-prefix=/ai/`（AI 服务下公网，只走内部 `AI_SERVICE_URL`）。
- **鉴权**：OAuth 自动建号经注册门 + 邮箱归一化；停用账号在 `requireUser` 实时拦截；demo 登录改显式 opt-in；缺失的 Prisma 迁移补齐（幂等）。
- **数据完整性（出图管线）**：regenerate 原子抢占 + 仅终态可重生；旧版本"新图成功后才删"；部分重试保留兄弟尺寸；合规移出 watchdog（不再把已出图判失败）；edit 保留源尺寸 + 上传换对象存储。
- **权限/IDOR**：跨 workspace 示例素材归属校验；审批需先 SUBMITTED、submit 不可回退已批准/终稿。
- **复查质量**：视觉复查带上已确认品牌规则。

### 5. Phase-1 收尾安全（本次新增，不留给下一棒）
- **关 demo 后门**：CDS env `AUTH_ALLOW_DEMO=0`，免密登录彻底关闭（之前线上仍开着 = 烧 key 漏洞）。
- **边缘门禁中间件**（`apps/web/src/middleware.ts`）：整站非公开路由（除 `/login`、`/api/auth/*`、`/api/health`、静态）都要 Auth.js 会话 cookie，公网无会话连接口都到不了。
- **密钥 + JWT 混合**：JWT 会话（登录身份）+ 边缘门禁 + OpenAI key 用 `SETTINGS_ENC_KEY` 加密存 DB、永不下发客户端、仅 worker 服务端使用。只有 `ADMIN_EMAILS` 白名单（= 用户邮箱 inernoro@gmail.com）能登录，注册关闭。
